知识管理与输出的工具使用探索

  Mac Liu     |    2020/03/28 posted in      工具分享     |    #高效软件 #Markdown

我的使用需求是什么？

在这个信息爆炸的时代，每天都接触到各种渠道过来的信息。而且我本身对于各种新鲜事物比较感兴趣，愿意去探索和尝试一些新的思维、工具，并不断地去优化我自己的工具箱。在这个过程中，我学会了使用Markdown语法，并记录一些笔记和思考，希望能有个地方能输出和分享。
经过一连串分析之后，我的需求逐渐清晰：

基本需求

  1. 在Mac（主力使用平台）、iOS和Win10上跨平台使用Markdown进行记录和写作；
  2. 为工作记录产生的Markdown文件需要能够转成合适的格式（PDF、docx）在同事之间进行分享；
  3. 为生活兴趣产生的Markdown文件需要能够在互联网上公开传播。

高效需求

  1. 在工具内提供分类管理和标签管理及搜索功能；
  2. 从写作到发布的工作流集成程度较高；
  3. 平台内容的可迁移性。

工具箱1.0版

在明确自己的基本需求之后，我尝试了几款在网络上使用度较高的平台和软件，但最终使用了一款高颜值的跨平台Markdown编辑器Typora。颜值高不高，其实你点进官网就能感受到了。
对于基本需求来说，Typora是一个很好的工具。以至于现在即便切到MWeb上的时候，我还会偶尔使用它作为写作工具。
为了配合Typora，我还有其他几个小工具辅助完成工作。
1. OneDrive云盘。 感谢公司为我提供了Office 365的全部应用，国内商用版的访问速度很快。我把所有的Markdown文件都放在OneDrive的文件夹中，同时也把Typora导出的PDF、docx文件都放在另一个文件夹中。这样操作的好处是我的所有原始文件都依赖OneDrive做了多设备同步和版本管理，另一方面是给所有同事可以直接分享OneDrive的链接。

福利：对于目前没有Office 365的同学，其实国内一直有一个很便宜的获取Office 365全套服务的方式。搜索“Office 365家庭版”，通常大家会凑齐6个人一起拼团。一年不到50的费用，你说香不香？

2. 图床工具。 什么叫图床工具？就是把本地图片上传到云端存储，以减少在分享过程中大量图片的传输。例如在刚才云盘介绍过程中，我的Markdown写作页面显示实际上这样子的。

现在在用的是PicGo和七牛云。在这篇文章中我不多讲如何配置图床工具了。

3. 基于Github的博客工具。 你现在看到的这个博客就是建立在Github的博客页面上的，这是为了满足输出和分享的需求。在初期，我是使用的Jekyll + Github Pages的方式建立了这个静态的博客页面。通常还有另一个方案是Hexo + Github Pages。这两个方案对于代码小白来说略显繁琐，因为你要学会使用：

  • 在本地电脑上搭建Jekyll或Hexo的编译环境；
  • 使用git向Github中提交变更；
  • 使用本地的代码编辑器（如：VScode）

工具箱2.0版

对于工具箱2.0，更多是围绕MWeb做的调整。
使用MWeb替代Typora之后，多了分类管理和标签管理，更方便了我对于知识的管理和搜索。在基本需求的满足上，MWeb其实也有缺点：不支持Windows平台。但是其提供的Markdown导入功能，可以让我在Windows平台继续用Markdown写作，然后在手机或者Mac端进行快捷导入。与它带来的优点相比，这一个小的妥协是我可以接受的。
在工具箱2.0中配合MWeb的工具有哪些？
1. iCloud云盘。作用其实与OneDrive类似。但在iOS和Mac上有更好的集成性。对于Windows的用户来说，依然可以使用OneDrive。
2. 图床工具。这里和工具箱1.0版其实是一样的。值得提到的是，其实MWeb中内置了图床上传工具，并不需要单独再使用PicGo。我没有用到这个功能，主要还是因为我想尽量保证图片是在云端存储，而不是在文档库中。

3. 基于Github的博客工具。这里和工具箱1.0版最大的不同是，MWeb替你集成了所有的工作。编辑网站样式、从文档库生成网站、发布网站只需要轻轻一点就完成了。完全不用切换到其他的工具上，整个流程也变得更加简洁和高效了。

有哪些工具没有进入工具箱？

没有永远的工具。个人需求在演进，工具也在演进。当这两个因素向不同的方向演进时，这个工具就可能不适合了。
印象笔记。老实说，印象笔记是我最早接触的知识管理软件。在2010年的时候，印象笔记和联通搞了一次送印象分的活动。我当时发动了身边所有有联通手机号的同学帮我薅羊毛，以至于到现在为止账户上的印象分还能支撑3年的高级会员兑换。印象笔记拥有很方便的信息收集功能，任何网上的信息在手机端通过分享或者微信的方式都能够导入，为什么我放弃了在上面继续进行知识管理呢？最主要的原因有两个：一是分享功能鸡肋；二是Markdown功能支持不好。
有道云笔记。和印象笔记类似。因为印象笔记已经有高级会员了，所以这个软件我使用得很少。
OneNote。受众面太窄，适合于工作圈内的同事分享。在国内有点水土不服。另外就是格式编辑上是个大问题。
幕布。暂时来看，幕布并不支持太多的格式，它更适合于结构化的文档，类似于思维导图。其实我偶尔也还在用它，因为它也支持将思维导图导出为Markdown文件。
石墨文档。太有工作气息了，是个简版的Office 365。适合于轻量化的协作场景。在我现在的需求中并未包含到需要多人协作来做知识管理，但在工作中我还是会和外部的同事、客户使用石墨文档来做信息沟通。
熊掌记。熊掌记其实比印象笔记能更符合我的基本需求一些。最终放弃它的原因是每个月都要收费。
语雀。还有没特别深度的使用过。
